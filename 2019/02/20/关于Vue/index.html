关于Vue

  1. 数据驱动DOM是Vue.js的核心理念，所以不到万不得已时不要主动操作DOM，只需要维护好数据，DOM的事Vue会优雅的处理。

  2. 数据与视图的双向绑定。

  3. 虚拟DOM：
    Virtual DOM并不是真正意义上的DOM，而是一个轻量级的JavaScript对象，在状态变化时，virtual dom会进行diff运算，来更新只需要被替换的dom，而不是全部重绘。

  4. vue-router，什么算是SPA呢？就是在前后端分离的基础上，加一层前端路由。前端路由，即由前端来维护一个路由规则。一种是利用锚点#，另一种就是HTML5的history模式，它使url看起来像普通网站那样，以“/”分割，但页面并没有跳转。

  5. 组件（Component）是Vue.js最核心的功能，也是整个框架设计最精彩的地方。
    可以说没见过的自定义标签就是组件，每个标签代表一个组件，在任何使用Vue的地方都可以直接使用。

  6. 第三方js受欢迎很重要的一个原因是因为对DOM操作进行了优化甚至是隐藏。
    JQuery号称是DOM操作的利器，将DOM封装成JQ对象并扩展了API，而MV框架取代JQuery的原因是因为在DOM操作这条路上做得更绝，直接屏蔽了底层操作，将数据映射到模板上。
